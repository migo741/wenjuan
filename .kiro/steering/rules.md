# Kiro Coding Rules

## 基本原则

- 只修改用户明确指定的文件
- 不允许重构无关代码
- 不允许引入新的第三方依赖
- 不允许删除已有功能
- 回答问题必须使用中文

## 技术栈约束

- 前端：Vue 3 + TypeScript
- 禁止使用 JSX（除非明确说明）
- 状态管理只能使用 Pinia

## 编码风格

- 使用 Composition API
- 函数必须是纯函数（除非涉及 IO）
- 禁止 any
- 必须补充必要的中文注释

## 行为约束

- 修改前先说明「你准备做什么」
- 如果需求不明确，必须先提问
- 不允许一次性修改超过 3 个文件

## 输出要求

- 输出 diff 或明确说明改动点
- 不要输出无关解释
